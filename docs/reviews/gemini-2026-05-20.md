# Gemini Cross-Check — agent-takkub Codex Review (2026-05-20)

## TL;DR
Overall verdict: **85% Agreement with Codex.** Codex identified the most critical architectural "trust but don't verify" boundaries. I found **3 additional blind spots**, including a **P0 project-leakage bug** in `_auto_trust` that breaks multi-tab safety, and a **P1 Thai text corruption** issue in the terminal rendering pipeline. The transition to multi-tab is currently "leaky" at the orchestrator level.

## Codex Findings Verification

### [Codex P0 #1] cwd validation gap
- **Verdict:** **AGREE**
- **Verification:** Read `orchestrator.py:1137` (assign) → `spawn()` (line 839). The code accepts `cwd` from the CLI and uses it directly: `spawn_cwd = cwd or default_cwd_for_role(...)`. There is no check against `projects.json[project].paths`.
- **Severity:** **P0** — A compromised teammate pane (e.g. via prompt injection) or a confused Lead could write to ANY directory on the system since Claude runs with `--dangerously-skip-permissions`.
- **Fix critique:** Codex's fix is solid. Validation must happen in `spawn()` to catch both `assign` and direct `spawn` calls.

### [Codex P0 #2] Lead `BLOCKED_DIRS` is prompt-only
- **Verdict:** **AGREE**
- **Verification:** `_render_lead_context()` (line 453) only generates a string. `spawn()` (line 906) passes `--dangerously-skip-permissions`. No executable hooks are present in the code to block writes.
- **Severity:** **P0** — Undermines the primary safety guarantee of the Lead role.
- **Fix critique:** Agreed. Using a local `settings.json` with `deny` rules for Lead is the most robust path.

### [Codex P1 #3] Lead context can inject the wrong project's `BLOCKED_DIRS`
- **Verdict:** **AGREE**
- **Verification:** `_render_lead_context()` (line 470) calls `active_project()` which reads the *globally* active project. If a user switches tabs while a Lead is spawning in the background, the wrong paths are blocked.
- **Severity:** **P1** — Breaks isolation in multi-tab workflows.
- **Fix critique:** Pass `project_ns` into `_render_lead_context(project)` as Codex suggested.

### [Codex P1 #4] Recent-exit resume state is keyed only by role
- **Verdict:** **AGREE**
- **Verification:** `self._recent_exits` is defined at line 594 as `dict[str, dict]` (role -> state). It is not namespaced by project.
- **Severity:** **P1** — Causes resume collisions (wrong `--continue` target) when the same role exists in multiple projects.
- **Fix critique:** Use `f"{project}::{role}"` as the key.

### [Codex P1 #5] Peer-to-peer CC can silently drop the Lead audit copy
- **Verdict:** **AGREE**
- **Verification:** `send()` (line 1273) checks `if lead and lead.session and lead.session.is_alive`. If Lead is down, it just logs an event and returns `True`. The audit is lost.
- **Severity:** **P1** — Lead's audit trail is "best effort," which contradicts the team coordination model.
- **Fix critique:** Fail-visible is better. If CC can't be delivered, the `send` should report it to the caller.

### [Codex P2 #6] Auto-routing behavior lives in prompt/docs
- **Verdict:** **AGREE**
- **Verification:** Search for "propose-then-fire" in `src/` returned 0 results in executable logic. It exists only in `CLAUDE.md`.
- **Severity:** **P2** — High risk of "hallucinated routing" or skipping the confirm gate under context pressure.
- **Fix critique:** Extracting to a `routing_planner.py` module is the correct engineering path.

### [Codex P2 #7] MCP write tools are pre-allowed for every Claude pane
- **Verdict:** **PARTIALLY-AGREE**
- **Verification:** `shared_dev_tools.py:104` includes `pms_create_task` etc. in `permissions.allow`.
- **Severity:** **P2** — Widens blast radius, but currently a "convenience vs security" trade-off.
- **Fix critique:** Role-specific MCP configs (Lead=RW, Teammate=RO) is a clean fix.

## Blind Spots (Gemini identified)

### [New P0] `_auto_trust` Project Leakage (Cross-tab Interference)
- **File:** `src/agent_takkub/orchestrator.py:1117`
- **Issue:** `_auto_trust(role_name)` calls `self.panes.get(role_name)`. Because `self.panes` is a property that resolves to the **currently active project tab**, switching tabs while a new pane is spawning will cause `_auto_trust` to target the **wrong project's pane** (or fail if the role isn't open in the new tab).
- **Why codex missed:** Codex looked at the logic of the method but missed the "leaky" dependency on the globally active UI state (`active_project()`).
- **Fix:** Pass `project_ns` to `_auto_trust(role, project)` and use `self._project_panes(project)`.

### [New P1] Thai Text Corruption on Chunk Boundaries
- **File:** `src/agent_takkub/terminal_widget.py:92`
- **Issue:** `write_bytes(data: bytes)` calls `data.decode("utf-8", "replace")`. If a multi-byte Thai character is split across two PTY read chunks (common in streaming), the decoding will produce a replacement character () for the partial sequence, corrupting the terminal display.
- **Why codex missed:** Requires knowledge of encoding edge cases in streaming terminal output.
- **Fix:** Forward raw bytes to `xterm.js` and decode in JS, or use a stateful `IncrementalDecoder` in Python before passing to `runJavaScript`.

### [New P2] Silent Failure on Port Bind Conflict
- **File:** `src/agent_takkub/cli_server.py:34`
- **Issue:** `listen()` raises `RuntimeError`, but `app.py` (assumed) likely doesn't catch it gracefully, or the UI doesn't show *why* the CLI is unresponsive.
- **Why codex missed:** It's an edge case of local environment setup.
- **Fix:** Add a clear error dialog in `MainWindow` if `CliServer` fails to bind.

## Rollout Strategy

### Phase 1: Security & Scoping (Critical)
- **Fix P0 #1 (cwd validation):** Bind `spawn()` to project paths.
- **Fix New P0 (_auto_trust leakage):** Namespace `_auto_trust` by project.
- **Fix P1 #3 & #4 (project scoping):** Namespace `_recent_exits` and pass project to context rendering.
- **Fix P1 Thai corruption:** Fix the decoding pipeline in `TerminalWidget`.

### Phase 2: Enforced Policy
- **Fix P0 #2 (Lead write guard):** Implement executable `settings.json` deny rules for Lead.
- **Fix P1 #5 (Durable CC):** Ensure Lead CC is either delivered or reported as failed.

### Phase 3: Robustness & DX
- **Fix P2 #6 (Auto-routing):** Move routing logic from prompt to Python.
- **Fix P2 #7 (MCP scoping):** Role-based MCP permissions.

---

## Bottom line
**Multi-tab isolation is the weakest link.** Priority #1 is fixing `_auto_trust` and `_recent_exits` project namespaces to prevent background tasks from "bleeding" between tabs.
