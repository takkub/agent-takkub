## 🔴 MUST-FIX (Release Blockers)

### 1. [Correctness] Non-Claude branch ละเลย `TAKKUB_TEAMMATE_MODEL`
ใน `src/agent_takkub/spawn_engine.py` ฝั่ง **non-claude branch** (`if effective_provider != CLAUDE:`) การคำนวณ `provider_model` ดึงค่ามาจาก `role_model_for` หรือ `model_for` เท่านั้น โดย **ไม่มีการเช็ค `os.environ.get("TAKKUB_TEAMMATE_MODEL")`** เลย
สิ่งนี้ทำให้:
- Provider อื่นๆ (Kimi, Cursor, Gemini, Codex) จะไม่สนใจ global env var นี้
- ขัดกับ requirement ย่อย (ข) ที่ระบุว่า `TAKKUB_TEAMMATE_MODEL=""` ต้องแปลว่า "ไม่ส่ง `--model`" เพราะใน non-claude branch หากกำหนด env เป็นค่าว่าง มันก็จะไป fallback หา role model / provider default ตามปกติ แทนที่จะส่ง model ว่าง
*(ส่วน claude branch จัดการลำดับความสำคัญและค่าว่างได้ถูกต้องแล้ว)*

### 2. [Correctness] Latch `seen_working` พังหาก Task ทำงานเสร็จไวมาก (Fast Task Regression)
ใน `orchestrator.py` ฟังก์ชัน `_check_idle_teammates` จะตั้งค่า `entry["seen_working"] = True` ก็ต่อเมื่อ watchdog ตื่นขึ้นมาพบว่า pane อยู่ในสถานะ **ไม่ได้อยู่ที่ ready prompt** (`if not pane.session.is_at_ready_prompt():`)
- **ช่องโหว่ (Regression):** หากมี task ที่รันเสร็จไวมากๆ (เช่น ภายในเศษเสี้ยววินาที) และกลับมาที่ ready prompt ก่อนที่ watchdog จะตื่นมาทำงานรอบแรก (tick time) watchdog จะเห็นว่า `is_at_ready_prompt()` เป็น True ทันที
- ผลที่ตามมาคือ โค้ดจะข้ามการ set `seen_working = True` ไปเลย และตัวแปรนี้จะเป็น False ตลอดกาล ทำให้ **pane ที่ทำงานเสร็จเร็วแต่ลืม `takkub done` จะไม่ได้รับ auto-reminder ตลอดกาล**

## 🔵 Nit / ข้อสังเกตอื่นๆ

### 6. [Cross-platform] `creationflags=SUBPROCESS_NO_WINDOW`
ใน `src/agent_takkub/provider_install.py` มีการส่ง `creationflags=SUBPROCESS_NO_WINDOW` ให้กับ `subprocess.run`
แม้ว่าบน macOS ตัวแปร `SUBPROCESS_NO_WINDOW` จะถูกนิยามให้เป็น 0 (ผ่าน `_win_console.py`) และ Python 3.9+ บน POSIX จะยอมรับ `creationflags=0` โดยไม่ raise `ValueError` แต่ถ้าใช้กับ Python เวอร์ชันเก่ากว่า 3.9 มันจะพังทันที โปรเจคนี้น่าจะใช้ Python ใหม่กว่า 3.9 เป็นหลักจึงรอด แต่ถือเป็น code smell เล็กน้อยที่ส่ง Windows-only parameter ลงไปตรงๆ
