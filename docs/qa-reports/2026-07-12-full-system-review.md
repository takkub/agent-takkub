## 🟠 HIGH SEVERITY

### 1. bootstrap_default_profile uses only .credentials.json to detect an existing login — never true on macOS (Keychain) → can rmtree a real logged-in profile

- **Where:** `src/agent_takkub/user_profile.py:466`  ·  **Subsystem:** auth-creds  ·  **Category:** cross-platform
- **Verdict:** CONFIRMED (high confidence)
- **Problem:** The 'is this a real profile, never touch it' guard at line 466 checks `(dest / _BOOTSTRAP_MARKER).exists() or (dest / ".credentials.json").exists()`. On macOS Claude Code stores its OAuth credentials in the login Keychain (service "Claude Code-credentials"), NOT in `<config_dir>/.credentials.json` — this exact fact is already handled elsewhere in the repo (limit_status._read_keychain_credentials, doctor.py darwin branch). So on macOS the `.credentials.json` fallback is dead: a real, logged-in installed profile that lacks the `.bootstrap-complete` marker is judged 'torn/stray' and line 468 `shutil.rmtree(dest, ignore_errors=True)` silently wipes it (settings.json, agents/, skills/, commands/, cloned session transcripts). The marker guard normally saves the common path, but any dir created before the marker feature existed — or a profile the user manually `claude login`-ed under CLAUDE_CONFIG_DIR before cockpit's first bootstrap — has no marker, and on macOS also no `.credentials.json`, so it is destroyed on next boot.
- **Failure scenario:** macOS installed build. User (or an older cockpit version) set up ~/.agent-takkub/claude-config with custom agents/settings and ran `claude login` (token → Keychain, no .credentials.json written), but the dir has no .bootstrap-complete marker. Next launch: dest.exists() True, marker absent, .credentials.json absent → line 468 shutil.rmtree wipes the user's real profile config silently.
- **Suggested fix:** On darwin, treat a Keychain entry as proof-of-login before rmtree: reuse `from .limit_status import _read_keychain_credentials` and, if `sys.platform == 'darwin' and _read_keychain_credentials()` returns non-None, return False (never touch).
