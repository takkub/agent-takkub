# System Review - 2026-05-20

Scope reviewed: `src/agent_takkub/` orchestrator/CLI/routing/cockpit/pane lifecycle, `runtime/` MCP/session files, hook/install scripts, `.claude/agents/`, `projects.json`, and multi-project tab handling.

## Findings

### 1. Critical/Security - Loopback CLI server trusts unauthenticated JSON, bypassing `TAKKUB_ROLE`

**File:line:** `src/agent_takkub/cli_server.py:64`, `src/agent_takkub/cli.py:81`

The CLI role gate exists only in the `takkub` client. The TCP server accepts any localhost JSON line and dispatches `spawn`, `assign`, `close`, and `done` without checking a shared secret or caller role. Any local process can connect to the port from `runtime/port` and perform Lead-only pane actions, including spawning panes with arbitrary roles/cwds inside allowed project roots.

**Concrete fix: add a per-cockpit nonce and require it on every request.**

```diff
diff --git a/src/agent_takkub/config.py b/src/agent_takkub/config.py
@@
 PORT_FILE = RUNTIME_DIR / "port"
+AUTH_FILE = RUNTIME_DIR / "cli-auth-token"
@@
+def write_cli_auth_token(token: str) -> None:
+    ensure_runtime()
+    AUTH_FILE.write_text(token, encoding="utf-8")
+
+def read_cli_auth_token() -> str | None:
+    try:
+        return AUTH_FILE.read_text(encoding="utf-8").strip()
+    except OSError:
+        return None
```

### 2. Critical/Security - Real PMS bearer token is stored in plaintext runtime MCP files

**File:line:** `runtime/shared-mcp.json:7`, `runtime/lead-mcp.json:7`, `src/agent_takkub/shared_dev_tools.py:142`

### 3. High - Role/project strings become paths without validation, enabling traversal writes

**File:line:** `src/agent_takkub/config.py:186`, `src/agent_takkub/orchestrator.py:1395`, `src/agent_takkub/orchestrator.py:1639`

`role` and `project` values are accepted as free-form strings and later become filesystem path components under `runtime/agents/<role>`, `runtime/pending-lead-cc-<project>.json`, `runtime/sessions/<date>/<project>/...`, and vault mirror paths. Combined with direct TCP access or a malformed `projects.json`, names like `..\..\x` can write outside the intended runtime subtree.

**Concrete fix: normalize role/project identifiers at ingress and assert resolved paths remain under their base directory.**

```diff
diff --git a/src/agent_takkub/config.py b/src/agent_takkub/config.py
@@
+import re
+
+_SAFE_NAME = re.compile(r"^[a-z0-9][a-z0-9_-]{0,63}$")
+
+def validate_name(value: str, kind: str) -> str:
+    name = (value or "").lower().strip()
+    if not _SAFE_NAME.fullmatch(name):
+        raise ValueError(f"invalid {kind}: {value!r}")
+    return name
@@
 def agent_role_dir(role: str) -> Path:
@@
-    d = RUNTIME_DIR / "agents" / role
+    role = validate_name(role, "role")
+    d = (RUNTIME_DIR / "agents" / role).resolve()
+    base = (RUNTIME_DIR / "agents").resolve()
+    if d != base and base not in d.parents:
+        raise ValueError(f"role path escapes runtime: {role!r}")
```

Apply the same validation in `Orchestrator._resolve_project()`, `spawn()`, `send()`, `done()`, and when adding custom roles in the UI.

## Positive Notes

- `PtySession.spawn()` uses `subprocess.list2cmdline(list(argv))` with argument lists; I did not find direct `shell=True` command injection in pane spawn.
- Multi-tab routing is mostly project-scoped: `TAKKUB_PROJECT` is injected at spawn and `cli_server` passes `from_project` into orchestrator calls.
- Lead protection exists in two layers for normal CLI use: `takkub done` is blocked for Lead, and `orchestrator.close()` protects Lead unless `force=True`.
- `--strict-mcp-config` is used when cockpit-managed MCP config is supplied, reducing accidental user-level MCP bleed-through.

## Top 3 Actionable

1. **Add authenticated IPC and server-side role enforcement.** This is the highest leverage fix because it closes the direct bypass of every client-side `TAKKUB_ROLE` rule.
2. **Rotate the exposed PMS token and stop writing bearer tokens to repo-local runtime JSON.** The current token is plaintext in multiple files; even though ignored by git, it is still a credential leak on the workstation.
3. **Validate role/project names before any filesystem use.** This removes a class of traversal bugs across runtime agents, pending CC queues, session notes, and vault mirrors with one small shared helper.
